# 品質特性リファレンス(ISO/IEC 25010 + 業務システム拡張)

qa-test-analysis / qa-code-review / qa-test-viewpoint / qa-test-design-review が
非機能観点の網羅チェックに使う。qa-code-review 向けのコードレベル・チェックポイント
への翻訳は [code-review-viewpoints.md](code-review-viewpoints.md) にある。全部を毎回テストするのではなく、**対象機能に関係する特性を
選び、選ばなかった理由も残す**こと。

## ISO/IEC 25010 主特性

| 特性 | 見ること(例) |
|---|---|
| 機能適合性 | 機能の完全性・正確性・適切性 |
| 性能効率性 | 応答時間、スループット、リソース消費、大量データ時の劣化 |
| 互換性 | 共存性、他システム・他バージョンとの相互運用 |
| 使用性 | 操作の分かりやすさ、誤操作防止、アクセシビリティ |
| 信頼性 | 障害時の回復、可用性、フォールトトレランス |
| セキュリティ | 認証・認可、機密性、改ざん防止、否認防止 |
| 保守性 | モジュール性、解析性、テスト容易性 |
| 移植性 | 環境適応性、インストール性 |

## 業務システム向け拡張観点

ISO 25010 に含まれないが業務アプリ(特に会計・基幹系)で重要な観点。

| 観点 | 見ること |
|---|---|
| 計算正確性 | 丸め・桁あふれ・通貨換算・端数処理の一貫性 |
| データ整合性 | 集計値と明細の一致、二重登録防止、トランザクション境界 |
| 監査証跡 | 誰がいつ何を変更したか追跡できるか、ログの改ざん耐性 |
| 再実行性 | 同じ処理の再実行で結果が一致するか(冪等性)、途中失敗からの再開 |
| 期間境界 | 期首・期末・締め処理・遡及修正・年度切替での正しさ |
| 権限分離 | 職務分掌、承認フロー、権限昇格の防止 |
| 帳票一致性 | 画面・帳票・エクスポート(Excel/CSV/PDF)間の数値・書式の一致 |
| 外部出力再現性 | Excel出力等の再現性(書式・数式・文字コード・ロケール) |

## 使い方

1. 対象機能に対して上記の表を上から流し、「関係する / 関係しない / 不明」に分類する。
2. 「関係する」特性ごとにテスト観点を最低1つ挙げる。
3. 「関係しない」と判断した特性はその理由を1行で残す(レビュー時の反証対象になる)。
