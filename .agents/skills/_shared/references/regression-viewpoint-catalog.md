# 回帰テスト観点カタログ

過去不具合から導出した再発防止観点の蓄積場所。qa-defect-analysis が追記し、
qa-test-viewpoint / qa-test-design-review が参照する。**このファイルは育てる資産**。
セッションを重ねるごとにエントリを追加・更新すること。

## エントリの書式

```markdown
## <観点名(発生パターンを一言で)>

### 根拠
- BUG-XXXX
- BUG-YYYY

### 発生しやすい条件
- (条件を箇条書き)

### 横展開対象
- (同じパターンが潜んでいそうな機能・モジュール)

### 推奨テスト観点
- (テスト設計に組み込むべき観点を箇条書き)

### 最終更新
- 2026-07-03(セッション: xxx)
```

---

## サンプルエントリ: 期間境界での集計漏れ

### 根拠
- BUG-1234(サンプル)

### 発生しやすい条件
- 期首・期末・締め日を跨ぐデータ
- 比較年度の切替
- 遡及修正後の再集計
- 対象データ0件

### 横展開対象
- 集計処理全般、帳票出力、エクスポート機能

### 推奨テスト観点
- 期首・期末ちょうどのデータと前後1日のデータ
- データなし / 1件 / 複数件
- 修正データあり・なしでの再実行結果の一致

### 最終更新
- 2026-07-03(初期サンプル。実データで置き換えること)
